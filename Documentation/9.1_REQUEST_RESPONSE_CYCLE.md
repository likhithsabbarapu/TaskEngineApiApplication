# 9.1 - Request-Response Cycle (Complete Workflow)

## 🔄 How Data Flows Through the System

This document traces a complete request from user action to database and back.

## 📱 Scenario 1: Adding a New Task

### Step-by-Step Workflow

```
┌─────────────────────────────────────────────────────────────┐
│ STEP 1: USER ACTION (React Frontend)                        │
└─────────────────────────────────────────────────────────────┘

User Interface:
┌──────────────────────────────┐
│ Add a new task... [Enter]    │ ← User types "Buy milk"
└──────────────────────────────┘

JavaScript Code Executed:
const handleAdd = async (e) => {
  e.preventDefault();              // Prevent page reload
  if (!title.trim()) return;       // Validate not empty
  setLoading(true);               // Disable button
  
  try {
    const res = await api.post('/tasks', {
      title: "Buy milk",           // Data from input
      status: 'TODO',              // Default status
      description: ''              // Empty for now
    });
    
    setTasks([...tasks, res.data]); // Add to state
    setTitle("");                   // Clear input
  } catch (err) {
    alert("Error adding task");
  } finally {
    setLoading(false);             // Re-enable button
  }
}
```

---

```
┌─────────────────────────────────────────────────────────────┐
│ STEP 2: HTTP REQUEST (Browser to Backend)                  │
└─────────────────────────────────────────────────────────────┘

Browser sends:
────────────────────────────────────────
POST /api/tasks HTTP/1.1
Host: localhost:8080
Content-Type: application/json

{
  "title": "Buy milk",
  "status": "TODO",
  "description": ""
}
────────────────────────────────────────

What happens:
1. JavaScript (Axios) creates HTTP request
2. Sets headers (Content-Type: application/json)
3. Serializes data to JSON
4. Sends to backend (network: ~100-500ms)
5. Browser waits for response
```

---

```
┌─────────────────────────────────────────────────────────────┐
│ STEP 3: CONTROLLER RECEIVES REQUEST (Spring Boot)          │
└─────────────────────────────────────────────────────────────┘

Request arrives at:
@PostMapping
public ResponseEntity<TaskResponseDTO> createTask(
    @Valid @RequestBody TaskRequestDTO requestDTO) {
    
    // requestDTO contains:
    // { title: "Buy milk", status: "TODO", description: "" }
    
    logger.info("Creating new task: {}", requestDTO.getTitle());
    
    // Convert DTO to Entity
    Task task = taskMapper.toTask(requestDTO);
    
    // Call service layer
    Task savedTask = taskService.saveTask(task);
    
    // Convert back to DTO
    TaskResponseDTO response = taskMapper.toResponseDTO(savedTask);
    
    // Return response
    return ResponseEntity.ok(response);
}

Timeline at Controller:
┌─────────────────────────────────────┐
│ @Valid validates input              │ ~10ms
│ Validation passed ✓                 │
├─────────────────────────────────────┤
│ TaskMapper converts DTO → Entity    │ ~5ms
├─────────────────────────────────────┤
│ Service layer called                │ ~10ms
└─────────────────────────────────────┘
```

---

```
┌─────────────────────────────────────────────────────────────┐
│ STEP 4: SERVICE LAYER (Business Logic)                      │
└─────────────────────────────────────────────────────────────┘

TaskService.saveTask() called:

public Task saveTask(Task task) {
    // Set timestamps
    task.setCreatedAt(LocalDateTime.now());
    task.setUpdatedAt(LocalDateTime.now());
    
    // Business logic here:
    // - Validate title length
    // - Check for duplicates
    // - Set default values
    // - Calculate metadata
    
    // Delegate to Repository
    return taskRepository.save(task);
}

What Service Does:
✓ Business rule validation
✓ Data transformation
✓ Logging
✓ Transaction management
```

---

```
┌─────────────────────────────────────────────────────────────┐
│ STEP 5: REPOSITORY (Data Access)                            │
└─────────────────────────────────────────────────────────────┘

TaskRepository.save(task) called:

public interface TaskRepository extends JpaRepository<Task, Long> {
    // JpaRepository.save() is inherited
}

Behind the scenes:
1. Hibernate converts Task entity to SQL
2. Generates INSERT statement:
   
   INSERT INTO tasks (
       title, 
       description, 
       status, 
       created_at, 
       updated_at
   ) VALUES (
       'Buy milk',
       '',
       'TODO',
       2026-03-24 10:30:00,
       2026-03-24 10:30:00
   );

3. Executes query via JDBC
```

## 📋 Scenario 2: Updating Task Status

### Quick Workflow

```
User clicks dropdown: "TODO" → "IN_PROGRESS"
    ↓
handleStatusChange(id=1, newStatus="IN_PROGRESS")
    ↓
HTTP PUT /api/tasks/1
Body: { ..., status: "IN_PROGRESS" }
    ↓
TaskController.updateTask() receives
    ↓
Find existing task
    ↓
Set new status
    ↓
TaskService.saveTask()
    ↓
UPDATE tasks SET status='IN_PROGRESS' WHERE id=1
    ↓
Task returned with updated fields
    ↓
HTTP 200 OK with updated task
    ↓
setTasks() updates state
    ↓
Task card re-renders with new color (Orange instead of Red)
```

## 🚨 Error Handling Flow

### When Something Goes Wrong

```
Example: User clicks Add with empty title

Step 1: Validation
const handleAdd = async (e) => {
  if (!title.trim()) return;  // ← Early exit, no API call
  // ...
}
Result: No network request sent, instant feedback

Example: Backend unavailable

Step 1: Request sent
api.post('/tasks', data)

Step 2: Network timeout (no response after ~30s)
Browser waits...
Axios timeout triggers

Step 3: Error caught
catch (err) {
  console.error("Connection error:", err);
  alert("Error adding task. Check backend connection.");
}

Step 4: UI update
setLoading(false)  // Re-enable button
setIsConnected(false)  // Show offline badge
// User can retry
```

## 🎯 Key Takeaways

1. **Frontend initiates**: React component handles user input
2. **HTTP request sent**: Axios serializes data to JSON
3. **Backend receives**: Spring Controller validates
4. **Business logic**: TaskService processes data
5. **Database stores**: PostgreSQL persists data
6. **Response sent**: JSON returned via HTTP
7. **Frontend updates**: React state changes
8. **UI re-renders**: User sees changes
