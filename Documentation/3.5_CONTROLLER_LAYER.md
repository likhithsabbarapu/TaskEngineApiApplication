# 3.5 - Controller Layer - REST API Endpoints

## 🎯 Overview

The **TaskController** is the API gateway that:
- Receives HTTP requests from the React frontend
- Processes the requests
- Delegates business logic to the Service layer
- Returns JSON responses to the frontend

---

## 📊 Controller Architecture

```
Frontend Request
    ↓
@RestController (TaskController)
    ↓
HTTP Method Handler (@GetMapping, @PostMapping, etc.)
    ↓
@RequestMapping("/api/tasks") (Base URL routing)
    ↓
Call TaskService (Business Logic)
    ↓
Map Entity to DTO (TaskMapper)
    ↓
Return ResponseEntity<JSON>
    ↓
Frontend Receives Response
```

---

## Class Declaration Explained

```java
@RestController
```
**What**: Spring annotation that marks this class as a REST API controller
**Does**:
- Automatically serializes return values to JSON
- Returns JSON responses directly (not HTML)
- Adds `@ResponseBody` to all methods
**Without It**: Would need to manually convert objects to JSON

---

```java
@RequestMapping("/api/tasks")
```
**What**: Base URL path for all endpoints in this controller
**Meaning**:
- All endpoints start with `/api/tasks`
- Example: `@GetMapping` becomes `GET /api/tasks`
- Example: `@GetMapping("/{id}")` becomes `GET /api/tasks/{id}`
**Centralizes**: URL management in one place

---

```java
@RequiredArgsConstructor
```
**What**: Lombok annotation for dependency injection
**Does**:
- Generates a constructor with required fields
- Spring uses this to inject `TaskService` and `TaskMapper`
**Alternative**: Manual constructor (verbose)

```java
// Without @RequiredArgsConstructor:
public TaskController(TaskService taskService, TaskMapper taskMapper) {
    this.taskService = taskService;
    this.taskMapper = taskMapper;
}

// With @RequiredArgsConstructor:
// Constructor generated automatically ✓
```

---

```java
@CrossOrigin(origins = "http://localhost:5173")
```
**What**: Cross-Origin Resource Sharing (CORS) configuration
**Problem**: Frontend and backend on different ports/domains
- Frontend: `http://localhost:5173`
- Backend: `http://localhost:8080`
**Solution**: CORS allows frontend to make requests
**Without It**: Browser blocks requests with CORS error

---

```java
private final TaskService taskService;
private final TaskMapper taskMapper;
```
**What**: Injected dependencies
**Purpose**:
- `taskService` - Calls business logic methods
- `taskMapper` - Converts Entity ↔ DTO
**Dependency Injection**: Spring automatically provides instances

## 🔗 HTTP Endpoints (Complete API Reference)

### Endpoint 1: Get All Tasks (with Pagination)

```java
@GetMapping
public ResponseEntity<Page<TaskResponseDTO>> getAllTasks(
        @PageableDefault(size = 10, page = 0, sort = "id", direction = Sort.Direction.ASC) 
        Pageable pageable) {
    Page<Task> tasks = taskService.getAllTasks(pageable);
    Page<TaskResponseDTO> response = tasks.map(taskMapper::toResponseDTO);
    return ResponseEntity.ok(response);
}
```

**HTTP Request**:
```
GET http://localhost:8080/api/tasks?page=0&size=10&sort=id,asc
```

**Query Parameters**:
| Parameter | Value | Meaning |
|-----------|-------|---------|
| page | 0 | Page number (0-indexed) |
| size | 10 | Tasks per page |
| sort | id,asc | Sort by id in ascending order |

**Response**:
```json
{
  "content": [
    { "id": 1, "title": "Buy milk", "status": "TODO" },
    { "id": 2, "title": "Call dentist", "status": "DONE" }
  ],
  "totalElements": 25,
  "totalPages": 3,
  "currentPage": 0,
  "pageSize": 10
}
```

**Code Explanation**:
```java
@PageableDefault(...) 
// Default: 10 items per page, sorted by ID ascending

Page<Task> tasks = taskService.getAllTasks(pageable);
// Service returns a Page object with pagination info

tasks.map(taskMapper::toResponseDTO)
// Convert each Task entity to TaskResponseDTO
// Hides internal details, only shows what frontend needs

ResponseEntity.ok(response)
// HTTP 200 OK with the Page of DTOs
```

### Endpoint 5: Create New Task (POST)

```java
@PostMapping
public ResponseEntity<TaskResponseDTO> createTask(
        @Valid @RequestBody TaskRequestDTO requestDTO) {
    Task task = taskMapper.toTask(requestDTO);
    Task savedTask = taskService.saveTask(task);
    return ResponseEntity.ok(taskMapper.toResponseDTO(savedTask));
}
```

**HTTP Request**:
```
POST http://localhost:8080/api/tasks
Content-Type: application/json

{
  "title": "Buy groceries",
  "description": "Milk, eggs, bread",
  "status": "TODO"
}
```

**Request Body**:
```json
{
  "title": "Buy groceries",
  "description": "Milk, eggs, bread",
  "status": "TODO"
}
```

**Response** (HTTP 201 Created):
```json
{
  "id": 1,
  "title": "Buy groceries",
  "description": "Milk, eggs, bread",
  "status": "TODO",
  "createdAt": "2026-03-24T10:30:00Z",
  "updatedAt": "2026-03-24T10:30:00Z"
}
```

**Code Explanation**:
```java
@Valid @RequestBody TaskRequestDTO requestDTO
// Validates incoming JSON against TaskRequestDTO
// If invalid, returns 400 Bad Request

taskMapper.toTask(requestDTO)
// Converts DTO to Entity
// Prepares for database storage

taskService.saveTask(task)
// Saves to database
// Returns Task with generated ID

taskMapper.toResponseDTO(savedTask)
// Converts back to DTO for response

ResponseEntity.ok(...)
// HTTP 200 with created task
```

**Frontend Usage** (JavaScript):
```javascript
const res = await api.post('/tasks', {
  title: 'Buy groceries',
  description: 'Milk, eggs, bread',
  status: 'TODO'
});
// Response: Task object with ID
```

### Endpoint 6: Update Task (PUT)

```java
@PutMapping("/{id}")
public ResponseEntity<TaskResponseDTO> updateTask(
        @PathVariable Long id,
        @Valid @RequestBody TaskRequestDTO requestDTO) {
    Task existingTask = taskService.getTaskById(id);
    existingTask.setTitle(requestDTO.getTitle());
    existingTask.setDescription(requestDTO.getDescription());
    if (requestDTO.getStatus() != null) {
        existingTask.setStatus(requestDTO.getStatus());
    }
    Task updatedTask = taskService.saveTask(existingTask);
    return ResponseEntity.ok(taskMapper.toResponseDTO(updatedTask));
}
```

**HTTP Request**:
```
PUT http://localhost:8080/api/tasks/1
Content-Type: application/json

{
  "title": "Buy groceries (updated)",
  "description": "Added milk",
  "status": "IN_PROGRESS"
}
```

**Path Variable**:
```
id = 1
```

**Response** (HTTP 200 OK):
```json
{
  "id": 1,
  "title": "Buy groceries (updated)",
  "description": "Added milk",
  "status": "IN_PROGRESS",
  "createdAt": "2026-03-24T10:30:00Z",
  "updatedAt": "2026-03-24T11:00:00Z"
}
```

**Code Explanation**:
```java
taskService.getTaskById(id)
// Find existing task in database

existingTask.setTitle(...)
existingTask.setDescription(...)
existingTask.setStatus(...)
// Update fields from request

taskService.saveTask(existingTask)
// Save updated task
// Database timestamp auto-updates

ResponseEntity.ok(...)
// Return updated task
```

**Frontend Usage**:
```javascript
const res = await api.put(`/tasks/1`, {
  title: 'Buy groceries (updated)',
  description: 'Added milk',
  status: 'IN_PROGRESS'
});
// Updates task in database
```

## 🔍 Annotations Explained

| Annotation | Purpose | Example |
|-----------|---------|---------|
| `@GetMapping` | Handle GET requests | Get tasks |
| `@PostMapping` | Handle POST requests | Create task |
| `@PutMapping` | Handle PUT requests | Update task |
| `@DeleteMapping` | Handle DELETE requests | Delete task |
| `@PathVariable` | Extract from URL | `/tasks/{id}` → id parameter |
| `@RequestBody` | Extract from body | JSON data |
| `@Valid` | Validate data | Check required fields |
| `@Operation` | API documentation | Swagger description |

---

## HTTP Status Codes

| Code | Meaning | When Used |
|------|---------|-----------|
| 200 | OK | Get, Create, Update success |
| 201 | Created | Create new resource |
| 204 | No Content | Delete success |
| 400 | Bad Request | Invalid input data |
| 404 | Not Found | Task doesn't exist |
| 500 | Server Error | Unexpected error |

## 🔄 Complete Request-Response Cycle

### Example: Frontend Changes Task Status

```
1. Frontend (React)
   handleStatusChange(taskId=1, newStatus="IN_PROGRESS")
   
2. Axios Call
   api.put('/tasks/1', { ...task, status: "IN_PROGRESS" })
   
3. HTTP Request Sent
   PUT /api/tasks/1
   Body: { id: 1, title: "Buy milk", status: "IN_PROGRESS", ... }
   
4. Backend Receives
   TaskController.updateTask(id=1, requestDTO)
   
5. Validation
   @Valid checks TaskRequestDTO
   If invalid → 400 Bad Request
   
6. Business Logic
   taskService.getTaskById(1)
   existingTask.setStatus("IN_PROGRESS")
   taskService.saveTask(existingTask)
   
7. Database
   UPDATE tasks SET status='IN_PROGRESS' WHERE id=1
   
8. Response Created
   TaskResponseDTO with updated data
   
9. HTTP Response Sent
   200 OK
   Body: { id: 1, ..., status: "IN_PROGRESS", ... }
   
10. Frontend Receives
    setTasks() updates React state
    Component re-renders
    UI shows new status with color change
    
11. User Sees
    Task status changed from red to orange
    All in smooth animation
```

## 🎯 Key Controller Concepts

1. **Separation of Concerns**: Controller only handles HTTP, not business logic
2. **DTOs**: Hide internal entity structure from API consumers
3. **Dependency Injection**: TaskService and TaskMapper provided by Spring
4. **Exception Handling**: Caught by GlobalExceptionHandler
5. **Validation**: @Valid ensures data integrity
6. **CORS**: Allows frontend on different domain to access API
7. **Logging**: Every action logged for debugging
