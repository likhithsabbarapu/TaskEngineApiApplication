# 4.0 - API Endpoints Reference

## 🌐 Complete REST API Reference

All endpoints are hosted at: `http://localhost:8080/api/tasks`

---

## 📋 API Endpoint Summary Table

| # | Method | Endpoint | Purpose | Returns |
|---|--------|----------|---------|---------|
| 1 | GET | `/` | Get all tasks (paginated) | Page of tasks |
| 2 | GET | `/list` | Get all tasks (simple list) | Array of tasks |
| 3 | GET | `/{id}` | Get single task | Single task |
| 4 | GET | `/status/{status}` | Get tasks by status | Page of tasks |
| 5 | POST | `/` | Create new task | Created task |
| 6 | PUT | `/{id}` | Update task | Updated task |
| 7 | DELETE | `/{id}` | Delete task | No content |

## 1️⃣ GET All Tasks (Paginated)

### Endpoint
```
GET /api/tasks?page=0&size=10&sort=id,asc
```

### Description
Retrieves all tasks with pagination support. Useful for large datasets.

### Query Parameters
| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| page | int | 0 | Page number (0-indexed) |
| size | int | 10 | Items per page |
| sort | string | id,asc | Sort criteria (field,direction) |

### Examples

**Get first page (10 items)**
```
GET http://localhost:8080/api/tasks?page=0&size=10
```

**Get page 2, sorted by title**
```
GET http://localhost:8080/api/tasks?page=1&size=20&sort=title,asc
```

**Sort by status descending**
```
GET http://localhost:8080/api/tasks?sort=status,desc
```

### Response (HTTP 200 OK)
```json
{
  "content": [
    {
      "id": 1,
      "title": "Buy groceries",
      "description": "Milk, eggs, bread",
      "status": "TODO",
      "createdAt": "2026-03-24T10:30:00Z",
      "updatedAt": "2026-03-24T10:30:00Z"
    },
    {
      "id": 2,
      "title": "Call dentist",
      "description": "Annual checkup",
      "status": "DONE",
      "createdAt": "2026-03-23T15:45:00Z",
      "updatedAt": "2026-03-24T09:00:00Z"
    }
  ],
  "totalElements": 25,
  "totalPages": 3,
  "currentPage": 0,
  "pageSize": 10,
  "isFirst": true,
  "isLast": false
}
```

### Response Fields
| Field | Type | Meaning |
|-------|------|---------|
| content | Array | List of tasks on this page |
| totalElements | int | Total number of tasks |
| totalPages | int | Number of pages |
| currentPage | int | Current page number |
| pageSize | int | Items per page |

### Error Responses
```json
HTTP 400
{
  "status": 400,
  "message": "Invalid page number",
  "timestamp": "2026-03-24T10:30:00Z"
}
```

## 2️⃣ GET All Tasks (Simple List)

### Endpoint
```
GET /api/tasks/list
```

### Description
Retrieves all tasks as a simple array. No pagination overhead.

### Parameters
None

### Example
```
GET http://localhost:8080/api/tasks/list
```

```json

  {
    "id": 1,
    "title": "Buy groceries",
    "description": "Milk, eggs, bread",
    "status": "TODO",
    "createdAt": "2026-03-24T10:30:00Z",
    "updatedAt": "2026-03-24T10:30:00Z"
  },
  {
    "id": 2,
    "title": "Call dentist",
    "description": "Annual checkup",
    "status": "DONE",
    "createdAt": "2026-03-23T15:45:00Z",
    "updatedAt": "2026-03-24T09:00:00Z"
  },
```

## 5️⃣ POST Create New Task

### Endpoint
```
POST /api/tasks
Content-Type: application/json
```

### Description
Creates a new task in the database.

### Request Body
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| title | String | Yes | Task name (1-255 chars) |
| description | String | No | Task details |
| status | String | No | Initial status (default: TODO) |

### Example Request
```
POST http://localhost:8080/api/tasks
Content-Type: application/json

{
  "title": "Buy groceries",
  "description": "Milk, eggs, bread",
  "status": "TODO"
}
```

### Request Validation
| Field | Rules |
|-------|-------|
| title | Required, non-empty, max 255 chars |
| description | Optional, max 1000 chars |
| status | Optional, must be valid enum |

### Response (HTTP 201 Created)
```json
{
  "id": 1,
  "title": "Buy groceries",
  "description": "Milk, eggs, bread",
  "status": "TODO",
  "createdAt": "2026-03-24T10:30:00Z",
  "updatedAt": "2026-03-24T10:30:00Z"
}
```

### Error Responses

**Missing title** (HTTP 400)
```json
{
  "status": 400,
  "message": "Title is required",
  "timestamp": "2026-03-24T10:30:00Z"
}
```

**Invalid status** (HTTP 400)
```json
{
  "status": 400,
  "message": "Invalid status value",
  "timestamp": "2026-03-24T10:30:00Z"
}
```

### Frontend Usage (React)
```javascript
const newTask = await api.post('/tasks', {
  title: 'Buy groceries',
  description: 'Milk, eggs, bread',
  status: 'TODO'
});
console.log(newTask.data.id); // Returns: 1
```

## 6️⃣ PUT Update Task

### Endpoint
```
PUT /api/tasks/{id}
Content-Type: application/json
```

### Description
Updates an existing task.

### Path Parameters
| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| id | Long | Yes | Task ID to update |

### Request Body
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| title | String | Yes | Updated task name |
| description | String | No | Updated description |
| status | String | No | Updated status |

### Example Request
```
PUT http://localhost:8080/api/tasks/1
Content-Type: application/json

{
  "title": "Buy groceries (updated)",
  "description": "Added milk and eggs",
  "status": "IN_PROGRESS"
}
```

### Response (HTTP 200 OK)
```json
{
  "id": 1,
  "title": "Buy groceries (updated)",
  "description": "Added milk and eggs",
  "status": "IN_PROGRESS",
  "createdAt": "2026-03-24T10:30:00Z",
  "updatedAt": "2026-03-24T11:45:00Z"
}
```

### Error Responses

**Task not found** (HTTP 404)
```json
{
  "status": 404,
  "message": "Task not found with id: 999",
  "timestamp": "2026-03-24T10:30:00Z"
}
```

### Frontend Usage (React)
```javascript
const updated = await api.put(`/tasks/1`, {
  title: 'Buy groceries (updated)',
  description: 'Added milk and eggs',
  status: 'IN_PROGRESS'
});
// Task updated in database
```

## 7️⃣ DELETE Task

### Endpoint
```
DELETE /api/tasks/{id}
```

### Description
Permanently deletes a task from the database.

### Path Parameters
| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| id | Long | Yes | Task ID to delete |

### Example Request
```
DELETE http://localhost:8080/api/tasks/1
```

### Response (HTTP 204 No Content)
```
(Empty body - successful deletion)
```

### Error Responses

**Task not found** (HTTP 404)
```json
{
  "status": 404,
  "message": "Task not found with id: 999",
  "timestamp": "2026-03-24T10:30:00Z"
}
```

### Frontend Usage (React)
```javascript
await api.delete(`/tasks/1`);
// Task deleted from database
// Remove from UI
setTasks(tasks.filter(t => t.id !== 1));
```

## 🔄 Common Request/Response Patterns

### Pattern 1: List with Filtering
```
Frontend Request
├─ GET /tasks/status/TODO?page=0&size=10
Backend Response
├─ Array of TODO tasks (paginated)
Frontend Update
├─ Display filtered tasks
```

### Pattern 2: Create and List
```
Frontend Request
├─ POST /tasks (create)
Backend Response
├─ New task with ID
Frontend Action
├─ Show new task in list
```

### Pattern 3: Update and Display
```
Frontend Request
├─ PUT /tasks/1 (update)
Backend Response
├─ Updated task
Frontend Action
├─ Show updated status with animation
```

### Pattern 4: Delete and Remove
```
Frontend Request
├─ DELETE /tasks/1
Backend Response
├─ 204 No Content
Frontend Action
├─ Remove from list with animation
```

## 📊 Data Type Reference

### TaskStatus Enum
```
TODO
IN_PROGRESS
DONE
```

### Task Object
```javascript
{
  id: number,           // Unique identifier
  title: string,        // Task name (required)
  description: string,  // Task details (optional)
  status: TaskStatus,   // Current status
  createdAt: timestamp, // Creation time
  updatedAt: timestamp  // Last modification
}
```

### Page Object (Pagination)
```javascript
{
  content: Task[],      // Array of items
  totalElements: number,// Total count
  totalPages: number,   // Number of pages
  currentPage: number,  // Current page (0-indexed)
  pageSize: number,     // Items per page
  isFirst: boolean,     // Is first page
  isLast: boolean       // Is last page
}
```
