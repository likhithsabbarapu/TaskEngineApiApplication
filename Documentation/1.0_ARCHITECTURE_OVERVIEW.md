# 1.0 - Architecture Overview

## 🏗️ System Architecture

The Enterprise Task System is a **3-tier web application** with clear separation of concerns:

```
┌─────────────────────────────────────────────────────┐
│           FRONTEND (React/Vite)                     │
│    - UI Components (App.jsx)                        │
│    - State Management (React Hooks)                 │
│    - HTTP Client (Axios)                            │
│    - iOS-Inspired Design                            │
└────────────────────┬────────────────────────────────┘
                     │
                HTTP Requests/Responses
                (REST API)
                     │
         ┌───────────▼──────────────┐
         │  BACKEND (Spring Boot)   │
         │                          │
         │  ┌────────────────────┐  │
         │  │ Controller Layer   │  │
         │  │ (REST Endpoints)   │  │
         │  └─────────┬──────────┘  │
         │            │             │
         │  ┌─────────▼──────────┐  │
         │  │ Service Layer      │  │
         │  │ (Business Logic)   │  │
         │  └─────────┬──────────┘  │
         │            │             │
         │  ┌─────────▼──────────┐  │
         │  │ Repository Layer   │  │
         │  │ (Data Access)      │  │
         │  └─────────┬──────────┘  │
         │            │             │
         └────────────┼─────────────┘
                      │
                 JDBC/SQL
                      │
         ┌────────────▼──────────┐
         │  PostgreSQL Database  │
         │  (Task Table)         │
         └───────────────────────┘
```

## 🔄 Data Flow (Request → Response)

### Adding a Task (POST)

```
1. Frontend (React)
   └─ User types task title
   └─ Clicks "Add" button
   └─ handleAdd() called

2. HTTP Request
   └─ POST /api/tasks
   └─ Body: { title: "Buy groceries", status: "TODO" }

3. Backend (Spring Boot)
   └─ TaskController receives request
   └─ TaskService.createTask() called
   └─ TaskRepository.save() writes to DB
   └─ Task entity returned

4. HTTP Response
   └─ 201 Created
   └─ Body: { id: 1, title: "Buy groceries", status: "TODO", ... }

5. Frontend (React)
   └─ Response received
   └─ State updated with new task
   └─ UI re-renders showing new task
   └─ Input field cleared
```

### Getting All Tasks (GET)

```
1. Frontend
   └─ useEffect hook triggers
   └─ fetchTasks() called
   └─ Every 5 seconds (polling)

2. HTTP Request
   └─ GET /api/tasks

3. Backend
   └─ TaskController.getAllTasks() called
   └─ TaskService.getAllTasks() called
   └─ TaskRepository.findAll() queries DB
   └─ List of Task entities returned

4. HTTP Response
   └─ 200 OK
   └─ Body: Array of task objects

5. Frontend
   └─ Response received
   └─ setTasks() updates state
   └─ Connection status set to "Connected"
   └─ UI re-renders with tasks
```

### Updating Task Status (PUT)

```
1. Frontend
   └─ User selects new status from dropdown
   └─ handleStatusChange() called

2. HTTP Request
   └─ PUT /api/tasks/{taskId}
   └─ Body: { ...task, status: "IN_PROGRESS" }

3. Backend
   └─ TaskController.updateTask() called
   └─ TaskService.updateTask() called
   └─ TaskRepository.save() updates DB
   └─ Updated task returned

4. HTTP Response
   └─ 200 OK
   └─ Body: Updated task object

5. Frontend
   └─ State updated via setTasks()
   └─ UI shows new status with color change
```

### Deleting a Task (DELETE)

```
1. Frontend
   └─ User clicks delete button
   └─ setDeleteAnimation(id) - shows slide-out animation
   └─ After 300ms, handleDelete() called

2. HTTP Request
   └─ DELETE /api/tasks/{taskId}

3. Backend
   └─ TaskController.deleteTask() called
   └─ TaskService.deleteTask() called
   └─ TaskRepository.deleteById() removes from DB
   └─ 204 No Content response

4. HTTP Response
   └─ 204 No Content

5. Frontend
   └─ Task filtered from state
   └─ Stats updated
   └─ UI re-renders without task
```

## 🗄️ Data Model

## 🔄 Synchronization Flow

### Real-time Updates
```
Frontend polls every 5 seconds:

useEffect(() => {
  fetchTasks();
  const interval = setInterval(fetchTasks, 5000);
  return () => clearInterval(interval);
}, []);

This ensures:
- Fresh data every 5 seconds
- Connection status updated
- New tasks from other sources shown
- Offline detection immediate
```

## 🎯 Summary

The Enterprise Task System follows a **3-tier architecture** with:

1. **Frontend** - React with iOS design, manages UI state
2. **Backend** - Spring Boot API, handles business logic
3. **Database** - PostgreSQL, stores task data

Each tier is independent but communicates through well-defined interfaces (HTTP/REST). This separation allows:
- Easy testing of each layer
- Independent scaling
- Clear responsibilities
- Maintainability and extensibility
