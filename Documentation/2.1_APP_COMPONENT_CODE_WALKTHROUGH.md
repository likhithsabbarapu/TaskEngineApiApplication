## Effect Hook - Data Fetching

```javascript
useEffect(() => {
  fetchTasks();
  const interval = setInterval(fetchTasks, 5000);
  return () => clearInterval(interval);
}, []);
```

**What**: Side effect hook for fetching data
**When**: Runs once when component mounts
**Purpose**: Fetch tasks and set up polling

**Breaking It Down**:

```javascript
useEffect(() => {
  // Dependency array is empty [] = runs once on mount
```
- Empty dependency array `[]` means:
  - Runs after component first renders
  - Doesn't run again on every re-render
  - Perfect for initialization

```javascript
fetchTasks();
```
- Call fetchTasks immediately
- Get initial task list
- Update connection status

```javascript
const interval = setInterval(fetchTasks, 5000);
```
- Set up polling interval
- Fetch tasks every 5000ms (5 seconds)
- Keeps data fresh
- Detects connection status automatically

```javascript
return () => clearInterval(interval);
```
- Cleanup function (runs on component unmount)
- Stops the interval
- Prevents memory leaks
- Stops unnecessary API calls

```javascript
try {
  const response = await api.get('/tasks');
```
**What**: HTTP GET request
**Endpoint**: `/tasks` (expands to `http://localhost:8080/api/tasks`)
**Awaits**: Promise to resolve
**Response**: Contains task data from backend

**What Happens**:
1. Sends GET request to backend
2. Waits for response
3. Network request: ~100-500ms
4. Response arrives with task array

---

```javascript

```
**What**: Updates tasks state with response data
**Why**: Three fallback options (in order):
1. `response.data.content` - If paginated response
2. `response.data` - If direct array
3. `[]` - If both fail (empty array)

**Example Response**:
```json
{
  "content": [
    { "id": 1, "title": "Buy milk", "status": "TODO" },
    { "id": 2, "title": "Call dentist", "status": "DONE" }
  ]
}
```



**What**: Update task in state
**How**: Map over tasks array
**Logic**:
- If `t.id === id`: Update this task's status
- Otherwise: Keep task unchanged

**Example**:
```javascript

After: [{ id: 1, status: "IN_PROGRESS" }, { id: 2, status: "DONE" }]
```

## Status Info Helper

```javascript
const getStatusInfo = (status) => {
  const statusMap = {
```

## Filtered Tasks & Stats

```javascript
const filteredTasks = tasks.filter(t => 
  filterStatus === 'ALL' ? true : t.status === filterStatus
);
```
**What**: Filter tasks based on selected filter
**Logic**:
- If filter is 'ALL': Include all tasks
- Otherwise: Only include tasks matching selected status

**Example**:
```javascript
tasks = [
  { id: 1, status: "TODO" },
  { id: 2, status: "IN_PROGRESS" },
  { id: 3, status: "DONE" }
]

filterStatus = "TODO"
Result: [{ id: 1, status: "TODO" }]

filterStatus = "ALL"
Result: All three tasks
```

## Complete Code Flow Summary

```
1. Component Mounts
   ↓
2. useEffect fires
   ↓
3. fetchTasks() called → GET /api/tasks
   ↓
4. Response received → setTasks() updates state
   ↓
5. Component re-renders with tasks
   ↓
6. Polling starts (every 5 seconds)

User Interactions:
   
   Add Task:
   Form submit → handleAdd → POST /api/tasks → setTasks → Re-render
   
   Update Status:
   Dropdown change → handleStatusChange → PUT /api/tasks/{id} → setTasks → Re-render
   
   Delete Task:
   Delete click → Animation → handleDelete → DELETE /api/tasks/{id} → setTasks → Re-render
   
   Filter:
   Tab click → setFilterStatus → filteredTasks recalculates → Re-render
```
